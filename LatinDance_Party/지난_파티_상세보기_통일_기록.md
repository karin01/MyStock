# 지난 파티 상세보기 통일 기록

## 📅 작업 일자
2024년 12월 19일

## 🎯 문제 상황
지난 파티에서 "상세보기" 버튼을 누르면 현재 파티의 상세보기와 다른 형태로 표시되는 문제 발생

## 🔍 문제 분석

### **기존 차이점들:**

1. **🏢 층수 필드 표시 방식**:
   - **현재 파티**: `party.location` (예: "지하 1층")
   - **지난 파티**: `party.location` (예: "강남구 테스트 댄스스튜디오")

2. **🗺️ 주소 필드 표시 방식**:
   - **현재 파티**: `party.address` (상세주소)
   - **지난 파티**: `party.address` 또는 `region + location` 조합

3. **📅 날짜 표시 방식**:
   - **현재 파티**: `formatPartyDateRange(party)` (범위 표시)
   - **지난 파티**: `formattedDate` (단일 날짜)

4. **필드 순서 및 구성**:
   - **현재 파티**: 댄스 분류, 바 이름 등 추가 필드 포함
   - **지난 파티**: 기본적인 정보만 표시

## ✅ 개선 사항

### **통일된 필드 구성:**
1. **📍 지역**: 파티가 열리는 지역
2. **💃 댄스**: 댄스 분류 (살바, 키좀바, 쥬크 등)
3. **🏢 바 이름**: 파티가 열리는 바/장소 이름
4. **🗺️ 상세주소**: 정확한 주소 (복사/지도 버튼 포함)
5. **🏢 층수**: 건물 내 위치 (지하 1층, 2층 등)
6. **📅 일시**: 날짜와 시간을 하나로 통합 표시

### **일관된 표시 방식:**
- **날짜 표시**: `formatPartyDateRange(party)` 사용으로 범위 표시
- **시간 표시**: 시간과 날짜를 하나의 필드로 통합
- **주소 표시**: "상세주소"로 명확히 구분
- **버튼 기능**: 복사 및 지도 보기 기능 유지
