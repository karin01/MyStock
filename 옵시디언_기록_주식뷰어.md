# 주식 정보·차트 뷰어 기록

## 개요

주식 **티커 심볼**을 입력하면 해당 종목의 **기본 정보**와 **가격 차트**를 보여주는 Python 도구를 만들었음.

- **위치**: `G:\내 드라이브\KNOU\Somoim\Jungwon_Drive_Obsidian_Vault\Stock`
- **데이터 출처**: Yahoo Finance (yfinance 라이브러리, 무료·API 키 불필요)

## 구성 파일

| 파일 | 설명 |
|------|------|
| `stock_viewer.py` | 핵심 로직: 정보 조회, 히스토리 조회, 차트 저장 |
| `app_streamlit.py` | 웹 UI (Streamlit) — 브라우저에서 조회 |
| `requirements.txt` | 필수 패키지 목록 |
| `README.md` | 사용법 안내 |

## 사용 방법

### 1) 명령줄

```bash
cd "G:\내 드라이브\KNOU\Somoim\Jungwon_Drive_Obsidian_Vault\Stock"
pip install -r requirements.txt
python stock_viewer.py AAPL
python stock_viewer.py 005930.KS 1y
```

- 터미널에 요약 정보 출력
- 같은 폴더에 `차트_티커.png` 저장

### 2) 웹 UI (선택)

```bash
pip install streamlit
streamlit run app_streamlit.py
```

- 브라우저에서 티커 입력 후 정보·차트 확인

## 티커 참고

- 미국: `AAPL`, `MSFT`, `GOOG` 등
- 한국: `005930.KS`(삼성전자), `035420.KS`(네이버) — 코스피 `.KS`, 코스닥 `.KQ`

## 기술 요약

- **get_stock_info(ticker)**: 현재가, 시가총액, PER, 섹터 등
- **fetch_history(ticker, period)**: OHLC 과거 데이터 (period: 1d, 1mo, 1y 등)
- **draw_chart(ticker, period, save_path)**: matplotlib으로 차트 이미지 저장
- 에러 처리: 패키지 없음, 데이터 없음, 잘못된 티커 등 처리

## 주의사항

- yfinance는 비공식 API이며, 참고·학습용으로 사용.
- 투자 판단은 본인 책임.

## 2025-02-23: 보유 종목 현재가 미표시 버그 수정

### 문제
- 장이 열려 있는데도 보유 종목 중 일부(예: KG모빌리티, SGA솔루션즈, 펄어비스 등)의 현재가·평가금액·손익이 "-"로 표시됨
- `451060.KS`, `AMD`처럼 티커 형식으로 등록한 종목만 정상 표시됨

### 원인
- `portfolio_holdings.json`에 **종목명**(한글)으로 저장된 경우가 있음 (예: "KG모빌리티", "디렉시온 반도체 3배 ETF")
- `_현재가_및_통화_조회()`가 이 값을 그대로 `get_stock_info()`에 전달
- yfinance/pykrx API는 **티커 심볼**(예: `003620.KS`, `SOXL`)을 필요로 함
- 종목명을 티커로 변환하는 `resolve_to_ticker()`를 호출하지 않아 조회 실패

### 수정 내용
- **app_streamlit.py** `_현재가_및_통화_조회()` 함수 수정
  - `resolve_to_ticker(ticker)`로 종목명 → 티커 변환 후 조회
  - 예: "KG모빌리티" → "003620.KS", "디렉시온 반도체 3배 ETF" → "SOXL"

## 2025-02-25: 로그인/로그아웃 시 국내 거래 현황 리로드 방지

### 문제
- 로그인·로그아웃할 때마다 `st.rerun()`으로 전체 앱이 리로드됨
- 국내 거래 현황(거래대금 상위, 등락률 등)까지 매번 다시 불러와 로딩 시간이 길어짐

### 수정 내용
- **app_streamlit.py**: "내 보유 종목" 섹션 전체를 `@st.fragment`로 감쌈
  - 로그인·로그아웃·회원가입·구매 등록·삭제 시 해당 fragment만 리로드
  - 국내 거래 현황·지수 차트 등 무거운 영역은 리로드하지 않음
  - `st.rerun()` 호출 제거 → fragment 상호작용 시 자동으로 해당 fragment만 재실행
